# 写作风格指南 — 反AI味核心笔记

> 每次写文章前先读这一份。AI味的本质是"太工整、太完美、太可预测"。
> 真人写文章有松有紧，有偏题，有废话，有随性的幽默。

---

## 禁止清单（AI常见毛病）

### ❌ 结构类

- **不要用"一、二、三、四"分段标题**，这是最大的AI味来源
- 不要把文章分成等长的段落
- 不要每段都有明确的功能（引入/展开/过渡/总结）
- 不要开头先铺垫再切入——直接扔场景进去
- 不要写"挑战与未来展望"式的小标题
- **不要用"从A到B"构造虚假范围** — "从个人生活到社会现象""从厨房到整个行业"都不是真范围
- **段落不要太齐** — 每段3-5句正好？那就有问题。真人写段落长短不一

### ❌ 句式类

- 不要用"说到这个…"、"其实…"、"不得不说…"、"值得一提的是"
- 不要用"由此可见"、"综上所述"、"正如我们所看到的"
- 不要用排比句式堆叠（不仅…而且…更是…）
- 不要用反问句结尾做"点睛之笔"
- 不要用"这大概就是……吧"这种感悟句式
- 不要用"你有没有想过…"开头
- **不要堆AI高频词** — 凸显、不可忽视、值得关注、深刻启示、关键作用、奠定基础、标志着、意味着、反映出、在…的背景下、在…的浪潮中、日益、愈发、不容忽视
- **不要用"充当着""扮演着""被誉为"替代"是"** — "起着关键作用"="很重要"，直接说就行了
- **不要"不仅…而且…更"三连** — 排比三连是LLM签名式写法
- **不要用".xxx ¹"²"做尾注** — 公众号文章不是论文
- **不要写"有人说""有网友表示""专家认为"没有出处** — 模糊归因（Weasel Words）
- **不要用"为了…需要""由于…的原因""在…的时候"** — 直接说，不要绕
- **不要用"可能大概也许某种程度上"叠加限定** — 过度 hedging
- **不要用"本质上说""真正的问题是""关键在于"** — 假装切中要害的权威话术

### ❌ 情绪类

- 不要追求"工整的温暖"
- 不要结尾强行升华
- 不要每个段落都服务中心思想——放一两句闲笔进去
- 不要用"希望对你有所帮助""如您需要…"这种协作式聊天语气
- 不要用"好问题！你说得对！"这种谄媚语气
- **不要结尾写"未来充满希望""前景光明"** — 通用积极结尾，AI标配

### ❌ 格式类

- 不要在文中插入markdown表格
- 不要用粗体堆砌关键词
- 不要用太多感叹号
- 不要写"XX字"或"阅读时长X分钟"这类元信息
- **不要滥用破折号（——）** — 一段里出现两次以上就是问题
- **不要用 emoji 装饰标题/要点** — 🚀 💡 ✅ 这些是 AI 痕迹
- **不要写冒号+解释的列表** — "关键词：解释"这种结构

## 真人风格特征（写前必读）

### ✅ 段落

- **长短不一** — 一句话独立成段是正常的，放一段五句话也是正常的
- 有的段落就是一句吐槽，不推进叙事
- 段落之间不一定有逻辑过渡，读者能跟上

### ✅ 语气

- 口语感，像在跟朋友发微信
- 偶尔用"好吧"、"绝了"、"麻了"、"救命"等网络语气词
- 能自嘲，不端着
- 有主观判断，不是"有人认为A，也有人认为B"
- **可以说"我"** — 第一人称不是不专业，是真人对话

### ✅ 细节

- **具体场景**而非抽象概念："我家厨房那个用了三年的奶锅"
- **具体数字**："19.9块"比"很便宜"好
- **第一人称真实体验**："我去厨房翻了一遍，果然"
- 有真实来源（微博评论、新闻当事人原话等）
- **写具体感受，不写概括评价** — 不是"这事值得关注"而是"我凌晨三点刷到这条，躺床上想了半小时"

### ✅ 节奏

- 信息密度变化：一段塞满事实，一段空着只有一句话
- 叙述中插入突然的个人联想
- 结尾不收束，留个尾巴或突然截断
- **不要同义词轮换** — "用户/消费者/使用者/客户"在一个段落轮换反而假，同一个称呼重复说就行
- **能直接用"是"就用"是"** — 不用"充当着""扮演着""被誉为""被视为"

## 风格模板示例

好的开头（场景切入）：
> 今天翻了一下我家厨房的碗柜，数了数，大大小小23个不锈钢餐具。然后我拿了一块冰箱贴挨个吸。吸上了16个。

好的过渡（不工整）：
> 但你知道最离谱的是什么吗？

好的结尾（不升华）：
> 奶锅还在用，但今天煮完这锅面，决定下次买个搪瓷的。
> 反正也不贵。

## 去AI味对照表（humanizer 29条清单）

| 类别 | AI痕迹 | 真人写法 |
|------|--------|---------|
| 意义膨胀 | "标志着一次关键转折" | "2019年成立，主要做…" |
| 归因模糊 | "专家表示""据业内人士" | 直接说谁说的，不说就当自己的判断 |
| -ing假深度 | "反映出行业趋势… contributing to…" | 一句话说完，不加尾巴 |
| 广告调 | "坐落于""坐拥""坐落于…的腹地" | "在…地方" |
| 回避"是" | "充当着核心引擎" | "是核心" |
| "不仅…而且…" | "不仅是A，更是B" | 直接说B |
| 排比三连 | "A、B和C" | 说一个最关键的 |
| 同义词轮换 | "用户/消费者/使用者/受众/客户" | 统一叫"用户" |
| 虚假范围 | "从日常饮食到社交生活" | 只写一个场景 |
| 被动语态 | "不需要配置" | "你不用配置" |
| 破折号滥用 | 一段两个——以上 | 拆成逗号或句号 |
| 粗体堆砌 | 关键词加粗 | 不加粗 |
| 花引号 | "xxx" | "xxx"（直引号） |
| 协作语气 | "希望有所帮助""如需帮助" | 不说 |
| 谄媚语气 | "好问题！说得对！" | 直接说事实 |
| 填充废话 | "为了…需要""由于…的原因" | "为了…""因为…" |
| 过度限定 | "可能大概也许某种程度上" | "可能"或直接断言 |
| 积极结尾 | "未来充满希望" | 不结尾 |
| 连字词 | 数据驱动、端到端、跨部门 | 数据驱动、端到端、跨部门（不用连字符也行）|
| 说教话术 | "本质上""真正的问题是" | 直接给论点 |
| "让我们来" | "让我们来看看""先说…再谈…" | 直接开始讲 |
| 碎片化标题 | 标题+1句话重复标题 | 标题后直接内容 |
| 尾注标记 | "xxx ¹""xxx ²" | 直接写在正文里 |
| 空泛评价 | "这事值得深思" | "我刷到以后躺床上想了半小时" |
| "随着…的" | "随着互联网的发展" | 砍掉"随着"，直接说"互联网发展后" |
| 知识截止声明 | "据了解""根据现有资料" | 直接写 |
